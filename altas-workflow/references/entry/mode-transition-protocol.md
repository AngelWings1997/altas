# 跨模式切换协议

> 定义 ALTAS Workflow 特殊模式之间的切换触发条件、状态保存和组合规则。

## 模式切换触发条件

| 当前模式 | 切换信号 | 目标模式 | 说明 |
|----------|----------|----------|------|
| DEBUG | 根因已定位，需要修复代码 | FAST / sdd_bootstrap | 切换前输出根因摘要 |
| DEBUG | 修复后需要补测试 | TEST | 保留根因上下文 |
| REVIEW | 发现需要重构 | REFACTOR | 输出重构建议后切换 |
| REVIEW | 发现测试不足 | TEST | 输出测试缺口后切换 |
| TEST | 发现 Bug | DEBUG | 保留测试上下文 |
| TEST | 发现需要重构测试代码 | REFACTOR | 保留测试策略 |
| REFACTOR | 重构完成需要验证 | TEST | 保留重构范围 |
| PERF | 定位到具体代码瓶颈 | FAST / sdd_bootstrap | 保留性能基线数据 |
| MIGRATE | 迁移后需要验证 | TEST | 保留迁移范围和回滚方案 |
| DOC | 文档更新涉及代码示例 | FAST | 仅限小改动 |

## 状态保存模板

切换模式时，在检查点中保存以下信息：

```markdown
### 模式切换检查点

- **从**: [当前模式]
- **到**: [目标模式]
- **当前进度**: [简要描述已完成的工作]
- **恢复锚点**: [切换回来时从哪里继续]
- **关键上下文**: [目标模式需要的核心信息，如根因/测试缺口/重构范围]
- **待办**: [当前模式中尚未完成的事项]
```

## 模式组合规则

### 可叠加的模式

| 主模式 | 可叠加 | 场景 |
|--------|--------|------|
| DEBUG | TEST | 排查问题同时补测试 |
| REVIEW | TEST | 审查代码同时补测试 |
| REFACTOR | TEST | 重构时 TDD 驱动 |
| MIGRATE | TEST | 迁移后验证 |

### 不可叠加的模式

- DEBUG + REFACTOR：应先完成排查再重构
- PERF + DEBUG：应先定位瓶颈类型再决定路径
- DOC + REFACTOR：文档和重构应分开进行

### 叠加执行规则

当需要叠加模式时：
1. 主模式优先，副模式在主模式检查点间隙执行
2. 叠加模式不需要重新走 Spec 流程
3. 叠加模式的检查点可简化为 1-2 行状态更新

## 自动切换 vs 手动切换

### 自动切换（Agent 可自行决定）

- DEBUG → TEST（根因定位后补测试）
- REVIEW → TEST（审查发现测试不足）
- REFACTOR → TEST（重构后验证）

自动切换时，Agent 必须在检查点中说明切换原因，用户可回复"继续原模式"来拒绝切换。

### 手动切换（需用户确认）

- 任何模式 → REFACTOR
- 任何模式 → MIGRATE
- 任何模式 → PERF

手动切换时，Agent 提议切换并说明理由，等用户确认后再切换。
