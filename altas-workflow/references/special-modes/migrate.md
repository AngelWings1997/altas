# MIGRATE 模式 - 数据/版本迁移专项协议

**触发词**: `MIGRATE` / `迁移` / `数据迁移` / `版本升级`

---

## 何时使用

- 用户需要迁移数据（数据库迁移/数据结构变更）
- 用户需要升级依赖版本（ Breaking Changes）
- 用户需要迁移 API（接口版本升级）
- 用户需要迁移基础设施（如数据库迁移/云服务迁移）

**默认规模**: L（高风险操作，必须有完整计划和回滚方案）

---

## 首轮动作

1. **确认迁移类型**:
   - 数据迁移（数据库/数据结构）
   - 版本升级（依赖/框架/运行时）
   - API 迁移（接口版本升级）
   - 基础设施迁移（数据库/云服务）
2. **确认迁移范围**:
   - 影响哪些系统/模块？
   - 影响多少数据量？
   - 影响哪些用户？
3. **确认约束**:
   - 是否允许停机？停机窗口多长？
   - 是否需要灰度/渐进式迁移？
   - 回滚时间要求？

---

## 迁移流程

### 1) 风险评估（必须）

**输出风险评估报告**:

```markdown
## 迁移风险评估报告

### 迁移目标
<描述迁移的目标和范围>

### 影响面分析
| 系统/模块 | 影响类型 | 影响程度 | 备注 |
|-----------|----------|----------|------|
| <系统 1> | 数据变更/接口变更/... | 高/中/低 | ... |
| <系统 2> | ... | ... | ... |

### 风险清单
| 风险项 | 发生概率 | 影响程度 | 缓解措施 |
|--------|----------|----------|----------|
| 数据丢失 | 低 | 高 | 完整备份 + 校验 |
| 服务中断 | 中 | 高 | 灰度发布 + 快速回滚 |
| 数据不一致 | 中 | 高 | 双写验证 + 对账 |
| 性能下降 | 高 | 中 | 压测 + 性能预算 |

### 必须满足的前置条件
1. [ ] 完整备份已完成
2. [ ] 回滚方案已验证
3. [ ] 监控告警已就绪
4. [ ] 相关人员已通知
```

### 2) 制定迁移方案

**针对每个迁移类型，给出推荐方案**:

#### 数据迁移方案

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **一次性迁移** | 小数据量、允许停机 | 简单快速 | 停机时间长、风险集中 |
| **渐进式迁移** | 大数据量、不能停机 | 风险分散 | 复杂度高、需要双写 |
| **蓝绿迁移** | 关键业务、零容忍停机 | 可快速回滚 | 资源成本高 |

#### 版本升级方案

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **直接升级** | Breaking Changes 少 | 简单快速 | 风险集中 |
| **渐进升级** | Breaking Changes 多 | 风险分散 | 需要兼容多版本 |
| **并行运行** | 关键系统 | 可快速回滚 | 资源成本高 |

### 3) 制定回滚方案（必须）

**回滚方案必须包含**:

- **回滚触发条件**: 什么情况下触发回滚？
  - 错误率 > X%
  - 性能下降 > Y%
  - 数据不一致 > Z 条
  - 核心功能不可用
- **回滚步骤**: 详细的回滚操作清单
- **回滚时间目标**: RTO（Recovery Time Objective）
- **回滚验证**: 回滚后如何验证系统正常？

**回滚方案模板**:

```markdown
## 回滚方案

### 回滚触发条件
满足以下任一条件即触发回滚：
1. 错误率 > 5% 持续 5 分钟
2. P95 响应时间 > 1000ms 持续 10 分钟
3. 数据不一致 > 100 条
4. 核心功能不可用

### 回滚步骤
1. [步骤 1] - 预计耗时：X 分钟
2. [步骤 2] - 预计耗时：Y 分钟
...
**总回滚时间**: Z 分钟

### 回滚验证
1. [验证项 1] - 预期结果：...
2. [验证项 2] - 预期结果：...
```

### 4) 制定迁移计划

**计划必须包含**:

- **Checklist**: 原子迁移步骤（每步独立可验证）
- **验证策略**: 每步完成后如何验证
- **监控指标**: 迁移期间需要监控哪些指标？
- **人员分工**: 谁负责什么？

**门禁**: 计划必须获得用户 `[Approved]` 后方可执行

### 5) 预演迁移（强烈推荐）

**在正式迁移前，强烈建议在测试环境预演**:

- 使用生产数据的子集（或脱敏全量数据）
- 完整执行迁移计划
- 验证回滚方案
- 记录实际耗时和问题

**预演报告模板**:

```markdown
## 迁移预演报告

**预演日期**: <YYYY-MM-DD>
**预演环境**: <测试/预发>
**数据规模**: <数据量，如"10% 生产数据">

### 预演结果
- 迁移是否成功：是/否
- 实际耗时：X 分钟（计划 Y 分钟）
- 遇到的问题：
  1. [问题 1] - 解决方案：...
  2. [问题 2] - 解决方案：...

### 与计划的差异
| 步骤 | 计划耗时 | 实际耗时 | 差异原因 |
|------|----------|----------|----------|
| 步骤 1 | X min | Y min | ... |

### 改进建议
<基于预演经验，对迁移计划的改进建议>
```

### 6) 执行迁移

**执行纪律**:

```
对每个 Checklist 项：
1. 执行步骤
2. 验证结果（功能 + 性能 + 数据一致性）
3. 检查监控指标
4. 若正常 → 输出检查点，进入下一步
5. 若异常 → 暂停，评估是否触发回滚
```

**铁律**:
- 每步必须验证通过后方可进入下一步
- 监控指标异常时必须暂停评估
- 达到回滚触发条件时必须立即回滚，不得侥幸

### 7) 迁移后验证

**验证清单**:

- [ ] 功能验证：核心功能是否正常？
- [ ] 性能验证：性能指标是否达标？
- [ ] 数据验证：数据是否完整/一致？
- [ ] 监控验证：监控告警是否正常？
- [ ] 备份验证：是否已备份新状态？

**输出迁移报告**:

```markdown
## 迁移报告

**迁移日期**: <YYYY-MM-DD>
**迁移类型**: <数据迁移/版本升级/...>
**迁移目标**: <描述目标>

### 迁移结果
- 是否成功：是/否
- 总耗时：X 分钟（计划 Y 分钟）
- 是否触发回滚：是/否

### 验证结果
| 验证项 | 结果 | 备注 |
|--------|------|------|
| 功能验证 | PASS/FAIL | ... |
| 性能验证 | PASS/FAIL | ... |
| 数据验证 | PASS/FAIL | ... |
| 监控验证 | PASS/FAIL | ... |

### 问题清单
1. [问题 1] - 解决方案：...
2. [问题 2] - 解决方案：...

### 经验总结
- 哪些做得好
- 哪些需要改进
- 对下次迁移的建议
```

## 门禁逻辑

| 场景 | 处理方式 |
|------|----------|
| 迁移前未备份 | **禁止执行迁移**，必须先备份 |
| 迁移前未验证回滚方案 | **禁止执行迁移**，必须先验证回滚 |
| 迁移中达到回滚触发条件 | **立即回滚**，不得继续尝试 |
| 迁移后验证失败 | 回滚或修复（用户决定） |
| 未预演直接生产迁移 | 输出强烈警告，建议先预演 |

---

## 与其他模式协作

- **MIGRATE → DEBUG**: 迁移后出现异常，进入 DEBUG 模式排查
- **MIGRATE → REVIEW**: 迁移完成后，进入 REVIEW 模式审查迁移质量
- **MIGRATE → ARCHIVE**: 迁移经验沉淀，进入 ARCHIVE 模式生成迁移知识库

---

## 特殊场景处理

### 场景 1: 大数据量迁移（>1TB）

- 必须使用渐进式迁移，不得一次性迁移
- 必须分批次，每批次独立验证
- 必须设置速率限制，防止影响生产

### 场景 2: 零停机迁移

- 必须使用双写策略（同时写新旧系统）
- 必须使用灰度切换（逐步将流量切到新系统）
- 必须准备快速回滚方案（分钟级）

### 场景 3: 跨团队/跨系统迁移

- 必须建立统一指挥链（谁决策？）
- 必须建立沟通机制（如 Slack 频道/电话会议）
- 必须明确各方职责（谁负责什么？）

## 迁移检查清单（通用）

**迁移前**:
- [ ] 风险评估已完成
- [ ] 迁移方案已评审
- [ ] 回滚方案已验证
- [ ] 预演已完成（强烈推荐）
- [ ] 备份已完成
- [ ] 监控告警已就绪
- [ ] 相关人员已通知
- [ ] 迁移计划已获批

**迁移中**:
- [ ] 每步验证通过
- [ ] 监控指标正常
- [ ] 无回滚触发

**迁移后**:
- [ ] 功能验证通过
- [ ] 性能验证通过
- [ ] 数据验证通过
- [ ] 监控验证通过
- [ ] 迁移报告已输出
